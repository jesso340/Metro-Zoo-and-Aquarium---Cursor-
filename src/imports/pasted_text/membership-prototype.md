You are a senior product designer and frontend engineer. Your task is to design and build a high-fidelity, interactive prototype of a modern membership experience that maximizes user attachment, engagement, and long-term retention.

This is NOT a generic membership UI. It must incorporate behavioral UX principles inspired by social platforms (like dynamic feeds, personalization, and habit loops), but applied ethically to create real value and a sense of belonging.

The output should be a polished React-based prototype (preferably using Next.js + Tailwind or a clean component-based structure) that I can demo to stakeholders.

---

## 🎯 Core Objective

Design a membership experience that makes users feel:

* “This is valuable to me”
* “This fits my life”
* “I’m getting more out of this over time”
* “I want to come back regularly”

---

## 🧠 Required Experience Framework

The experience must follow this loop:
Trigger → Action → Value → Reflection → Return

Your UI and flows should clearly support each stage.

---

## 🧩 Core Screens to Design

### 1. Membership Home (Dashboard)

This is the MOST important screen.

It should function like a personalized feed and answer:

* What can I do right now?
* What’s new for me?
* What have I gained?

Must include:

* Personalized welcome ("Good evening, Jessica")
* Dynamic content/feed (events, perks, updates)
* “Use it now” CTAs
* Progress summary (e.g., perks used this month)
* Value reinforcement (e.g., “You saved $120 this year”)
* Recommended actions based on user behavior

---

### 2. Benefits & Perks Discovery

Make benefits easy and engaging to explore.

Include:

* Categorized perks (e.g., Events, Discounts, Experiences)
* Personalized recommendations section
* Filters and/or search
* Clear benefit cards with:

  * Title
  * Short value description
  * CTA (Redeem, Learn More, RSVP)

---

### 3. Progress & Membership Journey

Show users they are building something over time.

Include:

* Membership tier or level (if applicable)
* Progress bar or milestone tracker
* Usage stats (e.g., perks used, events attended)
* Achievements or badges (optional but encouraged)

---

### 4. Value Recap / Insights

Reinforce why the membership is worth it.

Include:

* Total savings or value gained
* Activity summary (monthly or yearly)
* Visualizations (charts or simple stats)

---

### 5. Community / Belonging Layer

Even if lightweight, this must exist.

Include:

* Member highlights or stories
* Social proof (e.g., “10,000 members like you”)
* Optional: upcoming group events or shared experiences

---

### 6. Notifications / Return Triggers

Encourage re-engagement without being spammy.

Include:

* Subtle notification center or alerts
* Examples:

  * “New event this weekend”
  * “You have 2 unused perks this month”
  * “New benefit added for you”

---

## 🎨 Design Requirements

* Clean, modern UI (inspired by Apple, Airbnb, or Stripe dashboards)
* Use spacing, hierarchy, and cards effectively
* Mobile-first but responsive for desktop
* Use neutral color palette with 1–2 accent colors
* Include realistic mock data

---

## ⚙️ Technical Requirements

* Use React (functional components)
* Prefer Tailwind CSS for styling
* Component-based architecture (reusable cards, sections, etc.)
* Include basic state to simulate personalization (mock user data)
* No backend required, but structure data cleanly as if it could scale

---

## 🧪 Interactions to Include

* Clicking a benefit opens a detail modal or page
* “Use” or “Redeem” actions give feedback (e.g., success state)
* Navigation between main sections (Home, Benefits, Progress, Community)
* Simulated dynamic updates (e.g., different recommendations)

---

## 🧠 Behavioral UX Principles to Reflect

* Visibility of value (always show what user is gaining)
* Personalization (content feels tailored)
* Progress (users feel momentum)
* Low friction (easy to explore and act)
* Occasional delight (small surprises or highlights)

---

## 🚫 What to Avoid

* Static, boring dashboard
* Generic “list of benefits” with no context
* Overly gamified or manipulative patterns
* Cluttered UI with no clear hierarchy

---

## ✅ Output Expectations

* Fully working frontend prototype
* Clean, readable code
* Thoughtful component structure
* Enough polish to present to a product/design team

---

## 💡 Bonus (if possible)

* Add a “What to do today” smart section
* Add light animations (hover states, transitions)
* Include a sample onboarding state vs returning user state

---

Build this as if it will be shown in a product review with designers, PMs, and engineers. Prioritize clarity, intentionality, and user value over flashy visuals.
